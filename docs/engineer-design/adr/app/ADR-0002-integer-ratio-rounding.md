# ADR-0002: 丸め計算は整数比に対する ceil/floor で行う

- **ステータス**: proposed
- **コンテキスト**: REQ-002 の丸め(単位×方向)を浮動小数点の誤差なしに満たす必要がある。金額計算での off-by-one(1円・1単位のずれ)は要件の数値例と食い違い、テスト・レビューで即座に不合格になる
- **決定ドライバー**: 正確性(要件の数値例との一致)、実装の単純さ
- **検討した選択肢**:
  - 案A: `perPerson = Math.ceil((total / people) / unit) * unit` — 利点: 直感的 / 欠点: `total / people` の中間値が非整数のとき浮動小数点表現になり、単位境界ちょうど付近での誤差リスクを個別に論証する必要がある
  - 案B: `perPerson = Math.ceil(total / (people * unit)) * unit` — 利点: 「整数 ÷ 整数」1回の比に対する ceil/floor のみ。total ≤ 9,999,999・people×unit ≤ 99,000 は倍精度で正確に表現され、商が整数のときは除算結果も正確な整数になるため、境界での誤差が構造的に生じない / 欠点: 式の意図に一言の説明が要る
- **決定**: 案B。除算を1回に減らし誤差の論証を単純化できる。floor 方向も同型(`Math.floor`)
- **帰結**: collected = perPerson × people、diff = collected − total はすべて整数演算となり誤差フリー / 実装者は式を分解・「改善」しないこと(案Aへの書き換えはこのADRの蒸し返しになる)
