# 企画案: 超小規模テストアプリ(開発パイプライン検証用)

## 使用した材料

- 人間のキックオフ指示(2026-07-17): テストプロジェクト / 完全無料ローンチ / 新規サービス登録なし(既存GitHubのみ) / SaaS・外部API不使用 / 静的Web(GitHub Pages配信) / スマホ対応 / 内容を人間が把握できる超小規模(1〜2画面)
- 市場調査サマリ: なし。本プロジェクトの目的は開発パイプライン(企画→要件→デザイン→設計→無人実装→ローンチ)の検証であり、市場需要の探索ではないため planner-research を省略した

## 提出案(3案)

### 案A: 割り勘計算機(推奨)

- 解決したい課題: 飲み会等の割り勘で端数処理・幹事調整を暗算するのが面倒でミスが起きる
- 対象ユーザー: プロジェクトオーナー本人と周辺(検証用)/ 提供価値(一言): 金額と人数を入れるだけで公平な割り勘が即出る
- 根拠となる調査事実: なし(パイプライン検証が目的。出典: キックオフ指示)
- 弱点・不確実性: 既存の類似ツールが多数あり独自性はない(検証目的のため許容)。需要根拠は未検証
- 推奨理由: **1画面・依存ゼロ・保存不要**で3案中最小。かつ端数処理(切り上げ/切り捨て/幹事負担)という**テスト可能な純粋ロジック**を含み、要件の受け入れ条件・テスト設計・レビューの各工程が形骸化しない

### 案B: 習慣カウンター

- 解決したい課題: 「今日◯◯を何回やったか」を覚えておけない(腕立て、水分補給など)
- 対象ユーザー: 同上 / 提供価値(一言): タップだけで今日の回数を記録・翌日自動リセット
- 根拠となる調査事実: なし(同上)
- 弱点・不確実性: localStorage永続化と日付リセットロジックの分だけ案Aよりスコープが大きい。需要根拠は未検証

### 案C: 呼吸法ガイド(4-7-8呼吸)

- 解決したい課題: リラックス呼吸法のカウントを自分で数えると集中できない
- 対象ユーザー: 同上 / 提供価値(一言): 画面のアニメーションに合わせて吸う・止める・吐くだけ
- 根拠となる調査事実: なし(同上)
- 弱点・不確実性: タイマー・アニメーション中心でロジックのテスト可能性が低く、QA工程の検証material としては弱い。需要根拠は未検証

## 発散で出した全案リスト(落とした理由を簡記)

1. 割り勘計算機 → 案Aとして提出
2. 習慣カウンター → 案Bとして提出
3. 呼吸法ガイド → 案Cとして提出
4. ポモドーロタイマー → タイマー+通知でスコープが1〜2画面制約に対しやや大
5. 単位換算ツール → ロジックが自明すぎてレビュー・QA工程の検証にならない
6. 文字数カウンター → 同上(ロジックが薄い)
7. マークダウンメモ → パーサが必要でスコープ大、依存ゼロ制約と相性が悪い
8. QRコード生成 → 生成ライブラリの同梱が必要で「内容を把握できる規模」を超える
9. サイコロ/意思決定ルーレット → ロジックがランダムのみでテスト設計が形骸化する
10. 1日で消えるTODO → 案B(日付リセット)とほぼ同型で重複
11. 買い物リスト → CRUD一式でスコープが3案より大きい
12. カラーパレット生成 → 対象ユーザー(本人)の利用シーンがない
13. じゃんけんゲーム → 検証目的でも成果物として残す意味が薄い
