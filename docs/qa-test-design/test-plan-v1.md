# テスト計画: 割り勘計算機(要件定義書 v1.1 基準)

## 独立性宣言

参照した資料: docs/pm-requirements/requirements.md v1.1 のみ。実装コード・設計ドキュメント(engineer-design / designer-ui)・実装者への問い合わせは行っていない。期待結果はすべて要件の受け入れ条件から導出した。

## 対象と対象外

- 対象: REQ-001〜005 の全受け入れ条件

## テスト観点一覧

- 計算の正しさ(High: 金額の誤りはアプリの存在価値を壊す×丸め境界は壊れやすい)
- 入力検証と状態表示(High: 誤結果の残留は誤った集金につながる)
- 境界値(High: 範囲端 1 / 9,999,999 / 2 / 99 とその外側)
- 公開・表示環境(Mid: 一度確認すれば劣化しにくい)

## テストケース

### TC-001: 基本の割り勘表示(High / REQ-001-1)
- 前提条件: アプリを開いた状態
- 手順: 合計金額 10000、人数 4 を入力
- 期待結果: 1人あたりの支払額が表示される

### TC-002: 即時再計算(High / REQ-001-2)
- 前提条件: TC-001 の入力済み状態
- 手順: 人数を 4 から 5 に変更
- 期待結果: ページ再読み込みなしで結果が1秒以内に更新される

### TC-003: 割り切れる場合の等分(High / REQ-001-3, REQ-003-2)
- 前提条件: アプリを開いた状態
- 手順: 合計 10000、人数 4、丸めの影響が出ない条件で確認
- 期待結果: 1人あたり 2,500円。差額は「ぴったり」に相当する表示

### TC-004: 丸め設定の選択肢と初期値(High / REQ-002-1)
- 前提条件: アプリを開いた直後
- 手順: 丸め単位・方向の選択肢と初期選択を確認
- 期待結果: 単位は 1円・10円・100円・500円・1000円 の5択、方向は 切り上げ・切り捨て の2択。初期値は 100円・切り上げ

### TC-005: 切り上げ丸めと過不足表示(High / REQ-002-2, REQ-003-1, REQ-003-2)
- 前提条件: アプリを開いた状態
- 手順: 合計 10000、人数 3、単位 100円、方向 切り上げ
- 期待結果: 1人あたり 3,400円、集金合計 10,200円、差額「200円余る」

### TC-006: 1円単位の切り捨てと不足表示(High / REQ-002-3, REQ-003-2)
- 前提条件: アプリを開いた状態
- 手順: 合計 10000、人数 3、単位 1円、方向 切り捨て
- 期待結果: 1人あたり 3,333円。集金合計 9,999円、差額「1円足りない(幹事負担)」

### TC-007: 不正値のエラー表示(High / REQ-004-1)
- 前提条件: アプリを開いた状態
- 手順: 次を順に試す: 金額 0 / 金額 -100 / 金額 3.5 / 金額 abc / 人数 1 / 人数 0 / 人数 2.5
- 期待結果: いずれも結果の代わりに不正である旨の表示が出る

### TC-008: 未入力は案内表示(High / REQ-004-2)
- 前提条件: アプリを開いた直後(両方未入力)
- 手順: 1. 初期表示を確認 2. 金額のみ 10000 を入力し人数は空のまま
- 期待結果: いずれもエラーではなく初期状態と同じ案内表示

### TC-009: 古い結果の残留なし(High / REQ-004-3)
- 前提条件: TC-005 の結果表示状態
- 手順: 1. 人数を 0 に変更 2. 続けて人数を空にする
- 期待結果: 1ではエラー表示、2では案内表示になり、いずれの時点でも以前の計算結果(3,400円等)が画面に残らない

### TC-010: 範囲の境界値・内側(High / REQ-001-1, REQ-004-1)
- 前提条件: アプリを開いた状態
- 手順: (金額, 人数) = (1, 2), (9999999, 99) を入力
- 期待結果: いずれも有効として結果が表示される

### TC-011: 範囲の境界値・外側(High / REQ-001-1, REQ-004-1)
- 前提条件: アプリを開いた状態
- 手順: 金額 10000000(上限+1)/ 人数 100(上限+1)を入力
- 期待結果: いずれも不正である旨の表示が出る

### TC-012: 公開URLでの動作(Mid / REQ-005-1)
- 前提条件: なし
- 手順: GitHub Pages の公開URLをブラウザで開き TC-001 を実施
- 期待結果: インストール・登録なしで動作する

### TC-013: スマホ幅での操作性(Mid / REQ-005-2)
- 前提条件: 画面幅375px相当の表示環境
- 手順: 全機能(入力・丸め選択・結果確認)を操作
- 期待結果: 横スクロールなしで全機能が操作できる

### TC-014: 外部通信ゼロ(High / REQ-005-3)
- 前提条件: ネットワーク監視が可能な環境(開発者ツール等)
- 手順: ページ読み込みから TC-005 の操作までのネットワーク要求を記録
- 期待結果: 公開URL自身のリソース以外への要求(CDN・API・フォント等)が0件

## 環境・データ準備要件

- ブラウザ(スマホ実機または375px相当のエミュレーション)、開発者ツール(ネットワーク監視)
- GitHub Pages の公開URL(REQ-005 前提)
- テストデータは各ケース記載の値のみ(外部データ不要)

## 検討したが対象外とした観点

- 同時実行・複数タブ: 要件に保存・共有機能がなく、タブ間で状態が独立するため
- 通信断・オフライン: 要件がオフライン動作(PWA)をスコープ外と明示しているため
- アクセシビリティ詳細(スクリーンリーダー等): 要件の受け入れ条件に定めがないため(要件欠落の可能性は低い: 検証用テストアプリの位置づけ。必要なら要件改訂を提案)
